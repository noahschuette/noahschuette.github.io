⌨️ Angular CLI

  • Commands

    ng new name --standalone false --routing false --skip-tests --dry-run Generates new workspace, removes standalone/routing/tests if needed. Dry-run only previews the possible output, without writing on disk

    ng serve --open Build & run testwise, ng build for deployment

    ng generate component path/name Generate name component in path, also works with modules, directives, classes, enums, pipes, services, etc.

    ng lint Check for warnings in files

    ng update Check for updates in modules, update certein modules

    ng deploy Automatically deploy to firebase / netlify / other services

  • Config Files

    angular.json: General config, manage starting point, global styles and assets

    .editorconfig: Overrides IDE formatting settings

    browserslist: Tells CLI with browsers should be supported with the project

    package.json: Manages 3rd party node.js & angular dependencies

    tsconfig.json, tsconfig.app.json, tsconfig.spec.json: Configure TypeScript

  • Libraries

    Inside a lib folder: ng-package.json configures library, public-api.ts to export files from the library. Any exported component/service/… can now be used in other projects

📜 TypeScript

  • Basics

    Type definition: type TypeName = { ObjectOrWhatever … }

    Generics: function functionName<T>(param1: T, param2: T) {}

    Interface: interface InterfaceName { key : Type, key2 : Type2, … } works as an implementable function (same as interface in java)

    Abstract classes work same as in java (with extends, super, etc.)

🏠 Modules & Standalone

  • Modules

    Holds one or more components, directives & pipes. Consists of the following variables in the header: declarations, to declare any component in the module. imports, to import shared modules into the module, providers: to provide services to the module and its component and bootstrap, to define the app starting point.

    Export components inside export array, or by providing them in child routing.

    Define Root-Module in main.ts: bootstrapModule(AppModule)

  • Standalone

    Alternative to Ng-Module. Set standalone = true on a component and then import other components/modules into it with imports = [] inside the component instead of the module

    Root-Component in main.ts: bootstrapApplication(AppComponent)

    Provide servies with @Injectable({providedIn: “root”}) , or in the providers array of a component

⛓️ Databinding

  • Input/Output

    @Input() nameA : type = xyz Add input, can be referenced inside template with [nameA]

    @Output() nameB : new EventEmitter<{name : type, ... }>(); Triggers Output that can be emitted in a function via this.nameB.emit({name: type, ... }); and referenced inside template with (nameA) or watched via this.nameB.subscribe( (name : type) ⇒ console.log(name) ) for example

  • References

    References starting with # can be added to the head of any HTML item and then be used inside the HTML template at any point, e.g. to get the value of an input field with #inputFieldName.value

    The # Reference can be viewed inside the TS file via ViewChild: @ViewChild(’referenceName’) varName : ElementRef; and used with this.varName.nativeElement.value for example

☝️ Directives

  • Common Directives

    *ngFor=”let item of list; let i = index” works as a for loop with item item and index i

    *ngIf=”condition else refrence” with #reference works as if for a HTML block, and else the reference block is shown.

    Note: Angular18 introduces @if, @for blocks instead of the directives

    [ngClass]=”{className: condition}” adds class className if condition is met

    [ngStyle]=”{styleAttribute: value}” sets style attribute to value (value is var in TS)

    [ngSwitch]=”value” can be used as switch with *ngSwitchCase and *ngSwitchDefault in children

  • Custom Directives

    A TS-class @Directive({ selector : ‘[directiveName]’}) can be added to any HTML element as [directiveName]class. The HTML element el can be referred to in the class via constructor (private el: ElementRef,private renderer: Renderer2) {} (generated via ng generate directive)

    @HostListener(’eventName’) funcName(eventData: Event) {} listens to event e.g. mouseenter on HTML element. Listen to document-wide events with ‘document:eventName’

    @HostBinding(’propertyName’) varName : type; refers certein property, e.g. style.backgroundColor which then can be set

🛎️ Services

  • General

    Injectable TS class. Optional requires @Injectable() header if the service is injected from other classes beside components/directives

    Provide services either to AppModule (same service-instance for whole application), AppComponent (same service-instance for all childs of the AppComponent), Single component (component has own instance). Add service to the providers array of the specific component/module, declare it inside the constructor for each .ts using the service!

🪧 Routing

  • General

    <router-outlet></router-outlet> in the app component html marks where the routing pages are set. Child routes also need an outlet on their parents component
    The module gets a
    RouterModule.forRoot(appRoutes) import where const appRoutes : Routes = [] contains the list of routes. (For child modules use RouterModule.forChild(appRoutes))

    Route example: {path: ‘static/:parameter’, component: SampleComponent, children : [] } Route at path loads component and has child routes

    Use route in link as [routerLink] instead of href. path is stored in an array

    routerLinkActive=”class” adds class to html element if route is active (for styling), use [routerLinkActiveOptions]=”{exact: true}” if needed

    Pass params and #-fragment:[queryParams]=”{keyName : ‘value’}”, [fragment]=”fragment”

  • Route programatically

    Navigate programatically: this.router.navigate([path], {relativeTo : [path2], queryParams: {}, queryParamsHandling : 'merge'/'preserve', fragment: fragment}); with router : Router

    Get route with params: private route : ActivatedRoute Fetch route parameter from this.route.params.subscribe((params) => {}) (same with queryParams & fragment)

  • 404, Guards, special cases

    404 Redirect: {path : ‘**’, redirectTo : ‘/not-found’, pathMatch: 'full' } as last route

    Guards (route params prevents routing front/back): canActivate : [GuardClass1], canActivateChild : [GuardClass1], canDeactivate : [GuardClass1]

    Pass static data: property data: {key : value} in route

    Pass dynamic data with resolver property: resolve : {key : ResolverClass} , load via this.route.data

  • Routing with Standalone

    Provide routing module in main.ts with bootstrapApplication(AppComponent, { providers: [ importProvidersFrom(AppRoutingModule)], });

👀 Async Data Handling

  • RxJS: Observables

    Requires RxJS library: npm install rxjs@6

    emit data/error-data in an interval / over time. can be subscribed & unsubscribed to this.subscription: Subscription = this.observable.subscribe( data ⇒ {}, error => {}, () => {onComplete...} );

    this.subscription.unsubscribe();

    Subscribe only once: this.oberservable.pipe(take(1)).subscribe()

    create observable with Observable.create( observer ⇒ { ... } ) and emit data with observer.next(data);

    On error emit error with observer.error(new Error(’msg’)). The observable dies then

    Complete a observable with observer.complete()

  • Signals (RxJS Subject replacement)

    Create signal with signalName = signal<type>(defaultValue); , read value in template from signalName()

    Use set(value) to set new value or update((oldValue) ⇒ oldValue+newValue) to replace based on old value

    Do something everytime a signal changes: e.g. doubleCounter = computed(() ⇒ this.signalName() * 2) or e.g. effect(() ⇒ console.log(this.signalName())

📝 Forms

  • Reactive Forms

    Create Form in TS: formName : FormGroup = new FormGroup({ ‘inputName’ : new FormControl(defaultValue, validators), … }) and then pass name to <form [formGroup]=”formName”> , pass <input formControlName=”inputName”> to each input included in the group.

    Get Form data in TS from formName.value , … in HTML from formName.get(’inputName’)

    Subscribe to value/status changes with formName.valueChanges.subscribe() , formName.statusChanges.subscribe()

    Form touched? formName.touched

    Set form value: formName.setValue({’inputName’ : ‘value’, … }) (Set only one part of the input with patchValue instead. Reset form with formName.reset

    Validators: Add them to a FormControl to make it valid or not (make it required, make it an e-mail form etc.)

    ng adds css classes .ng-invalid and .ng-touched if input is invalid and touched

⬇️ Pipes

  • General

    Can be used after a | in the HTML document, can also be used in *ngFor directives

    Pipe class: implements PipeTransform: transform(value : any, parameters) { … return value }
    & has parameter
    name : ‘pipeName’, and can be used after a | as pipeName:param1:param2

    Pipe has second property pure = false which makes it update to changes

    Use promises / observables in HTML document with the async pipe

🛌 REST-APIs

  • Example Request, Pipes

    Get private http : HttpClient in constructor. Use pipe and map to map the data, use catchError to catch error data. Define the type of the getter data after .get()

    fetchPosts() {
	let params = new HttpParams();	params = params.append('key','value');
	return this.http
		.get<{ [key : string] : Post}>( // request type (GET), type of data to get
			'url',
			{
				headers : new HttpHeaders({'key' : 'value'}), // set headers
				params : params, // set params
				observe : 'response', // request full response instead of data only
				responseType : 'json'
			}
		)
		.pipe(
			tap(event => ... ) // do something with data without changing it
			map(responseData => { ... return data }), // map response data
			catchError(errorResponse => { return errorData }) // do something with the error data
		);
}

    Best practice: Use HTTP in a service, then return this.http.post(url, body) inside a function and subscribe to the function as this.httpService.getSomething().subscribe( responseData => {...}, errorData => {...} )

  • Interceptors

    Interceptors: Manipulate any request before it leaves the server. Declare in module:providers: { provide : HTTP_INTERCEPTORS, useClass: InterceptorService, multi: true } . Create class InceptorService implements HttpInterceptor. Implement intercept(req: HttpRequest<type>, next: HttpHandler) { … return next.handle(req); }

✨ Optimization & Export

  • Lazy-Loading Modules

    Lazy-loading modules: (e.g. with parent-path path : ‘childModuleRouteName’ in ChildModule ) Remove the path-name as path : ‘’ and then add it back to the parent module, e.g. AppModule with path: 'childModuleRouteName', loadChildren: () => import('module-path').then(x => x.ChildModule) . Remove import of ChildModule from AppModule . In tsconfig.json: “module” : “ESNext”

  • Environment variables

    Environment variables: exported in file environments.ts and environment.prod.ts (for production) as key-value pairs. Imported from anywhere later

🧪 Testing

  • Example & General
    describe('AppComponent', () => { 
	beforeEach() => { // run before each test
		TestBed.configureTestingModule({ // header for the test
			declarations : [AppComponent], imports : [AppRouting], providers: [SomeService]
			// declare, import and provide same as in a module
		}
		TestBed.compileComponents();
	}
	
	it('test name'), waitForAsync() => {
		const fixture = TestBed.createComponent(AppComponent); // The component to test
		const instance = fixture.debugElement.componentInstance; // The instance of the component
		expect(instance).toBeTruthy(); // Do the test (here check if component loads)
	}
}

    Use fixture.detectChanges() if something changed

    Inject services with inject(), look out for subscribable task with spyOn()
